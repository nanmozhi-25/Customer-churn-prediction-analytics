Project Viva Guide (தமிழ் விளக்கம்)

Complete project explanation specifically structured for college presentations and external examiners.

1. Project oda Main Aim Enna? (What is this project?)

Intha project oda peru "Telecom Customer Churn Prediction System". Oru telecom company-la (like Jio, Airtel, Vodafone), endha customers ellam udane network-a vittu poga (Churn aaga) vaaipu iruku nu munnadiye AI & Machine Learning use panni kandupudikara project thaan idhu. Itha vechu company avangala retain panna offers kudukalam.

2. Idhula Namma Ennanna Use Pannirukom? (Tech Stack)

Ithu oru complete Full-Stack + AI project:

  • Frontend (UI): React JS (Vite) - Dashboard, Customer list, Charts ellam paakka (Glassmorphism design la pannirukom).
  • Backend (API): Node.js & Express JS - Frontend-kum Database-kum naduvula data paathu process panna.
  • Machine Learning (AI): Python (FastAPI) & Scikit-Learn - Customer churn aavangala mattangala nu predict panna. Namma Random Forest Algorithm use pannirukom.
  • Database: MongoDB - Customer details save panna. (And database fail aana local JSON file use aagura mathiri fallback vechirukom).

3. Ithu Epdi Work Aaguthu? (How it works? - Working Flow)

  • Data Input: User/Admin frontend dashboard la oru customer-oda details-ah enter pannuvanga (Example: Avanga ethana maasama plan use pandranga (Tenure), enna contract vachirukanga (Month-to-month ah), tech support iruka illaya nu). Illana bulk-a oru CSV file ah kooda upload pannalam.
  • Backend Gateway: Intha details React la irunthu Node.js server-ku pogum.
  • ML Prediction (The Brain): Node.js andha data-va Python ML Microservice-ku anuppum. Python-la namma train panni vechiruka ML model, intha customer poga evlo % risk iruku (High risk ah, Low risk ah) nu calculate panni result ah thiruppi anuppum.
  • Smart Fallback Engine (Highlight Point 🔥): Oruvela Python ML server run aagalana kooda namma system crash aagathu! Node.js laye oru "Heuristic Rules Engine" vechirukom. Athuve customer oda details vechu (eg: Month-to-month contract na risk athigam) manual ah predict panni system ah run panna vechudum.
  • Visualization: Kidaicha result-a React dashboard la azhagana charts ahavum, Power BI canvas la interactive reports ahavum display panniduvom. Itha vechu admin alert aagi customers-ku messages anuppalam.

4. Namma YEN intha technologies-a choose pannom? (Tech Choices - Deep Dive)

  • Frontend-ku React JS (Vite): Normal HTML/CSS use panratha vida, React use panna website romba fast-a irukum (Virtual DOM concept). Dashboard-la neraya charts & data dynamically maarite irukara nala, React thaan best. Vite use pannathala server load aaga fractions of seconds thaan aagum.
  • Backend-ku Node.js & Express: Node.js-oda main highlight "Non-blocking I/O". Adhavathu, ore nerathula 1000 per predictions kettaalum server hang aagathu, queue-la vechu fast-a process pannum.
  • AI/ML-ku Python (FastAPI): AI & Machine learning-ku Python thaan king! Ana Python-a Node.js kooda connect panna oru paalam (bridge) thevai. Adhukuthaan FastAPI use pannirukom. Ithu matha frameworks (like Flask/Django) vida romba fast.
  • Database-ku MongoDB: Ithu oru NoSQL database. Telecom data-la eppo vena puthusa oru column (example: 5G usage) add aagalam. SQL/MySQL na table structure-a mathite irukanum, aana MongoDB-la andha prechana illa (Flexible schema).

5. Machine Learning Model ULLA epdi work aaguthu? (How AI actually predicts?)

  • Namma Random Forest Algorithm use pannirukom. Namma kitta irukathu "Tabular Data" (Excel sheet mathiri iruka details). Intha mathiri data-ku Deep Learning-a vida Random Forest thaan highest accuracy kudukum.
  • Epdi Train pannom? Namma kitta pazhaya 7000+ customer data iruku (telecom_customer_churn.csv). Andha data-va ML model-ku kuduthu padikka vechom. Model kandupudicha main pattern: "Yaaralaam Month-to-month contract vechirukangalo, mathum Fiber Optic internet use panni bill athigama katrangalo... avanga udane network-a vittu poidranga".
  • Intha padichu vechikitta ariva (knowledge) oru file-a save panni vechirukom. Adhu thaan `.pkl` (Pickle) file. Python FastAPI andha `.pkl` file-a load panni thaan live-a predict pannuthu!

6. Namma Project-oda "Masterpiece" (Enterprise Fallback System 🔥)

Viva-la examiner kitta kandippa sollanum! Oruvela nethu rathiri Python server crash aayiduchu, illa MongoDB down aayiduchu... appo namma software velai seyyatha? Kandippa seyyum!

  • Database Fallback: MongoDB velai seyyalana, Node.js udane local-la iruka oru `local_db.json` file-la data-va save panna kuthichidum.
  • AI Fallback: Python ML server down aana, Node.js-kullaye namma ezhuthi vechiruka "Heuristic Rule Engine" (Manual javascript rules) automatically activate aagi, customer details-a vechu athuve thராயமா predict panni system crash aagama thaduthudum.

Idhu oru Enterprise/MNC level software architecture thinking! Itha sonneenga na kandippa impress aavanga.

💻 Proposed Frontend (UI) Architecture

தமிழ் விளக்கம் / Tamil Overview

முன்மொழியப்பட்ட Frontend வடிவமைப்பு React 18 மற்றும் Vite கொண்டு உருவாக்கப்பட்டுள்ளது. இது கூறு-அடிப்படையிலான (Component-driven) கட்டமைப்பைக் கொண்டுள்ளது. பயனரின் அமர்வு மற்றும் கருப்பொருள் (Auth & Theme) ஆகியவை Context API மூலம் நிர்வகிக்கப்படுகின்றன. Recharts வரைபடங்கள் மற்றும் லூசிட் ஐகான்கள் தடையற்ற காட்சி அனுபவத்தை வழங்குகின்றன.

The frontend layer uses a modular, component-based Single Page Application (SPA) architecture designed for high-frequency user interactions and visual data analysis:

  • State Management Layer: Utilizes React Context API (AuthContext) to propagate JWT tokens, user profiles, and active session states down the React tree without prop-drilling.
  • Component Taxonomy: Decoupled into Stateless Presentational Widgets (custom metrics cards, loaders, and glass containers) and Stateful Page Controllers (Dashboard, Sandbox, and Reports).
  • Data Visualization Routing: Employs Recharts dynamically bound to state buckets. Re-renders chart nodes instantly when filter slicers or pagination pages are updated.
  • Global Styling Tokens: Uses vanilla CSS custom properties (variables) defined in index.css to drive the Aurora AI Light aesthetic across all component frames.
[Frontend Stack] React 18 SPA + Vite + Recharts + Context API + CSS Variables + JWT Local Storage Persistence

⚙️ Proposed Backend (API) Architecture

தமிழ் விளக்கம் / Tamil Overview

பின்மொழியப்பட்ட Backend வடிவமைப்பு Node.js மற்றும் Express.js கொண்ட REST API நுழைவாயிலாக செயல்படுகிறது. இது பாதுகாப்பு, தரவு செயலாக்கம் மற்றும் AI மாடல்களுக்கான இணைப்பினை நிர்வகிக்கிறது. தரவுத்தளம் முடங்கினால் தானாகவே உள்ளூர் கோப்புக்கு மாறும் தற்காப்புத் திறன் இதில் உள்ளமைக்கப்பட்டுள்ளது.

The backend server implements a decoupled, controller-service-repository pattern acting as an API gateway, request broker, and resilience orchestrator:

  • API Gateway Router: Structured Express router group mapping clean REST endpoints. Implements CORS controllers to enable secure requests from authorized client origins.
  • Middleware Pipeline: Handles token verification (JWT checks), request validations, file uploads (Multer for CSV parsing), and uniform error logging.
  • Dynamic Failover Controllers: The database layer monitors Mongoose connection states. On connection loss, it transparently swaps query executions to local flat JSON storage.
  • Dual-Engine Inference Router: Connects to Python FastAPI on port 8000. If connection fails, it routes numeric customer features to the Javascript heuristic rules engine.
[Backend Stack] Node.js + Express.js + Mongoose ORM + JWT Session Auth + Bcrypt Credentials + Multer Parser

Schema Relationships

The platform utilizes a structured **MongoDB NoSQL schema design** managed via Mongoose models. Although MongoDB is schema-less by nature, Mongoose enforces strict validation, data-type binding, and enum checking:

  • User Collection: Stores system analysts with secure Bcrypt password hashes.
  • Customer Collection: Houses telecom subscriber demographic profiles and billing data.
  • Prediction Collection: Logs real-time ML risk predictions, referencing customers via customerId.
  • Report Collection: Tracks metadata, file paths, and compiler stages of compiled PDF reports.

Copyable DAX Analytical Measures

Create these quick measures inside the Power BI modeling ribbon to calculate key customer retention indicators:

// 1. Total Customers Count
Total Customers = COUNT(telecom_customer_churn[customerId])

// 2. Total Churned Customers
Churned Customers = CALCULATE(
    COUNT(telecom_customer_churn[customerId]), 
    telecom_customer_churn[churn] = "Yes"
)

// 3. System Churn Rate (%)
ChurnRate = DIVIDE([Churned Customers], [Total Customers], 0) * 100

// 4. Monthly Revenue Lost to Churn
Monthly Revenue Lost = CALCULATE(
    SUM(telecom_customer_churn[monthlyCharges]), 
    telecom_customer_churn[churn] = "Yes"
